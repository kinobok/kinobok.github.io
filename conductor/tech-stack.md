# Architecture
kinꚘbok uses a decoupled architecture with a statically hosted frontend that consumes a static JSON data file generated by a scheduled backend scraper.

# Frontend
- **Language:** TypeScript
- **Framework:** Next.js (React)
- **Map Rendering:** Leaflet (via react-leaflet)
- **Data Parsing:** PapaParse (for CSV processing)
- **Styling:** Vanilla CSS
- **Icons:** Lucide React
- **Testing:** Vitest

# Backend (Scraper)
- **Language:** Python 3.11+
- **HTTP Client:** HTTPX
- **HTML Parsing:** BeautifulSoup4
- **Data Validation & Modeling:** Pydantic
- **String Matching:** RapidFuzz
- **Testing:** Pytest

# CI/CD & Deployment
- **Automation:** GitHub Actions (daily scraper runs, formatting checks, deployment)
- **Hosting:** GitHub Pages
